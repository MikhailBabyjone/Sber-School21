# Принсипҳои барномасозии сохторӣ

Дар барномасозии сохторӣ ду принсипи асосӣ вуҷуд доранд:
- принсипи «тақсим кун ва ҳукмронӣ намо» (декомпозитсия) — роҳи ҳалли масъалаҳои мураккаб тавассути ҷудо кардани онҳо ба як маҷмӯи вазифаҳои хурдтар ва мустақил, ки фаҳмидан ва ҳал карданашон осонтар аст. Ҳангоми ҳалли чунин вазифаҳо такрори код кам мешавад ва роҳҳалҳои онҳо, агар лозим шавад, дубора истифода бурда мешаванд.
- принсипи тартиббандии иерархӣ — роҳи сохтани қисмҳои масъала дар шакли сохторҳои иерархии дарахтмонанд бо илова кардани ҷузъиёти нав дар ҳар сатҳ (аз сатҳи боло бо як нуқтаи вуруд то сатҳҳои поёнӣ бо сохторҳои мушаххаси додаҳо ва татбиқҳо). Яъне, дар як сатҳ набояд ҳамзамон ҳисобҳо ва амалиёти вуруд-барориш ҷой дошта бошанд. Ҳамин тавр, ҳангоми истифодаи сабки сохторӣ барнома мисли як «пироги қабат-қабат» аз боло ба поён сохта мешавад. Хатогиҳо дар сатҳҳои поёнӣ ба вуҷуд меоянд ва ба боло интиқол меёбанд, ки он ҷо ба корбар намоиш дода мешаванд.

Ташаккул ва рушди барномасозии сохторӣ бо номи Эдсгер Дейкстра алоқаманд аст.
* Принсипи 1. Аз истифодаи оператори гузариши бешарт goto даст кашидан лозим аст.
* Принсипи 2. Ҳар як барнома аз се сохтори асосии идоракунанда таркиб меёбад: пайдарпайӣ, шохагирӣ, давр.
* Принсипи 3. Дар барнома сохторҳои асосии идоракунанда метавонанд ба таври дилхоҳ дар якдигар ҷойгир шаванд. Ҳеҷ гуна воситаҳои дигари идоракунии пайдарпайии иҷрои амалҳо пешбинӣ намешаванд.
* Принсипи 4. Порчаҳои такроршавандаи барномаро метавон дар шакли зербарномаҳо (тартибот ва функсияҳо) таҳия кард. Ба ҳамин монанд (дар шакли зербарномаҳо) порчаҳои аз ҷиҳати мантиқӣ яклухти барномаро ҳам метавон таҳия кард, ҳатто агар онҳо такрор нашаванд.
* Принсипи 5. Ҳар гурӯҳи дастурҳои аз ҷиҳати мантиқӣ анҷомёфтаро бояд ҳамчун блок таҳия кард. Блокҳо пояи барномасозии сохторӣ мебошанд.
* Принсипи 6. Ҳамаи сохторҳои номбаршуда бояд як даромад ва як баромад дошта бошанд.
* Принсипи 7. Таҳияи барнома қадам ба қадам, бо усули «аз боло ба поён» (top-down method) анҷом дода мешавад.

Паёмадҳо ва иловаҳои принсипҳои дар боло зикршуда:

1. Манъи истифодаи тағйирёбандаҳои глобалӣ. Тағйирёбандаи глобалӣ ҳар тағйирёбандае ҳисоб мешавад, ки берун аз тани функсия эълон шудааст.

```cpp
int x1; // Запрещено
static int x2; // Запрещено

int sum(int a, int b) {
 int result = a + b; // Разрешено
 return result;
}
```

2. На бештар аз як баромад аз функсия. Истисно санҷиши пешакии аргументҳои функсия дар оғози тани функсия мебошад. Бо вуҷуди ин, агар имкон бошад, беҳтар аст бе ин санҷиш кор кард.

```cpp
int count_negative(int *arr, int n) {
 if (arr == NULL) return -1; // Хорошо, таким образом сигнализируем об ошибке
 int count = 0;
 for (int i = 0; i < n; i++)
 if (arr[i] < 0) count++;
 return count; // Хорошо, один выход
}

int min(int a, int b) {
 if (a < b) return a; // Плохо, должен быть только один выход
 return b;
}

int max(int a, int b) {
 int result = b;
 if (a > b) result = a;
 return result; // Хорошо, так как это единственный выход
}

int factorial(int n) {
 int result = 1;
 for (int i = 1; i <= n; i++) result *= i;
 if (n < 0) return -1; // Плохо, проверка аргументов должна быть строго в начале тела функции
 return result;
}
```

3. На бештар аз як баромад аз давр — ин метавонад ҳам шартҳои худи давр бошад ва ҳам калимаи калидии break дар ҳолати даври беохир. Ҳамчунин истифодаи калимаи калидии `continue` манъ аст, зеро рафтори он ба амалиёти `goto` монанд аст.

```cpp
#define ARRAY_SIZE 5

int main(void) {
 int pos_zero = -1;
 int arr[ARRAY_SIZE] = {1, 2, 0, 4, 5};
 for (int i = 0; i < ARRAY_SIZE; i++) // Первая точка выхода
 if (arr[i] == 0) {
 pos_zero = i;
 break; // Плохо, вторая точка выхода из цикла
 }

 for (int i = 0; i < ARRAY_SIZE && pos_zero == -1; i++) // Хорошо, единственная точка выхода
 if (arr[i] == 0) pos_zero = i;

 int i = 0;
 while (true) {
 if (i == ARRAY_SIZE || pos_zero != -1) break; // Хорошо, так как это единственная точка выхода
 if (arr[i] == 0) pos_zero = i;
 i++;
 }

 for (int i = 0; i < ARRAY_SIZE && pos_zero == -1; i++) // Единственная точка выхода
 {
 if (arr[i] != 0) continue; // Запрещено
 pos_zero = i;
 }

 return 0;
}
```

4. Дарундарунӣ доштани ҳама гуна блокҳо набояд аз 4 зиёд бошад. Ҳисобкунии дарундарунӣ аз худи функсия оғоз намешавад.

```cpp
#include <stdio.h>

#define MATRIX_COLS 3
#define MATRIX_ROWS 3

int main(void) {
 int matrix[MATRIX_ROWS][MATRIX_COLS];
 for (int i = 0; i < MATRIX_ROWS; i++)
 for (int j = 0; j < MATRIX_COLS; j++) {
 int element = matrix[i][j];

 // Подсчитываем сколько раз element встречается в матрице
 // Лучше вынести в отдельную функцию, чтобы минимизировать уровень вложенности
 int count = 0;
 for (int k = 0; k < MATRIX_ROWS; k++)
 for (int l = 0; l < MATRIX_COLS; l++)
 if (element = matrix[k][l]) // Плохо, 5-ый уровень вложенности
 count++; // Плохо, 6-ый уровень вложенности
 }
 return 0;
}
```

6. Манъи истифодаи функсияи қатъи фаврии барнома exit().

```cpp
int sum_array(const int *arr, int n) {
 if (arr == NULL)
 exit(1); // Плохо, лучше возвращать код ошибки и обрабатывать её извне, чем прерывать программу
 int result = 0;
 for (int i = 0; i < n; i++) result += arr[i];
 return result;
}
```
